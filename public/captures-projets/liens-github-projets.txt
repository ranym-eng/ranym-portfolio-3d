## Use cases (summary)

### User (base role)

**Test manager** and **Administrator** inherit these capabilities.

- **Register** — sign up
- **Authenticate** — sign in
- **View profile** — account information
- **Lock the application** — session lock / idle protection
- **View test traces** — browse test trace data
- **View statistics** — metrics and summaries

### Test manager

- All **User** capabilities
- **Manage log files** — log file administration

### Administrator

- All **User** capabilities
- **Manage serial numbers** — device / serial reference data
- **Manage operations** — operational configuration or tracking
- **Receive notifications** — system or business alerts
- **Manage users** — user lifecycle and permissions

# Gymify Web

## Système de Gestion de Salle de Sport

Gymify Web est une application web développée dans le cadre des cours de l’École d’Ingénieurs ESPRIT pour l’année universitaire 2024-2025.

L’application permet de gérer plusieurs salles de sport à travers une plateforme centralisée destinée aux administrateurs, responsables de salles, entraîneurs et sportifs. ## Aperçu du projet

Gymify Web vise à simplifier la gestion quotidienne des salles de sport en proposant une interface moderne, intuitive et réactive.

L’application permet notamment de :

* gérer plusieurs salles de sport depuis une seule plateforme ;
* administrer les utilisateurs selon leurs rôles ;
* gérer les activités, abonnements, cours et événements ;
* organiser les plannings des entraîneurs ;
* permettre aux sportifs de consulter les offres, participer aux événements et effectuer des paiements ;
* gérer les produits, avis, blogs et réclamations ;
* intégrer des services externes comme Stripe, Google Auth, Google Calendar, reCAPTCHA, la météo et DeepInfra.

## Configuration

### Variables d’environnement

Créer un fichier `.env.local` à partir du fichier `.env` ou `.env.example`, puis renseigner les variables nécessaires à votre environnement local.

Exemple :

```env
DATABASE_URL="mysql://root:@127.0.0.1:3306/gymifyweb?serverVersion=8.0"
STRIPE_SECRET_KEY="your_stripe_secret_key"
STRIPE_PUBLIC_KEY="your_stripe_public_key"
GOOGLE_CLIENT_ID="your_google_client_id"
GOOGLE_CLIENT_SECRET="your_google_client_secret"
RECAPTCHA_SITE_KEY="your_recaptcha_site_key"
RECAPTCHA_SECRET_KEY="your_recaptcha_secret_key"
DEEPINFRA_API_KEY="your_deepinfra_api_key"
```

### Base de données

1. Démarrer XAMPP.
2. Lancer Apache et MySQL.
3. Créer une base de données MySQL.
4. Mettre à jour la variable `DATABASE_URL` dans le fichier `.env.local`.

Créer la base de données avec Symfony :

```bash
php bin/console doctrine:database:create
```

Exécuter les migrations :

```bash
php bin/console doctrine:migrations:migrate
```

# Système de Prédiction du Risque Cardiaque

Application Streamlit permettant de démontrer plusieurs modèles de Machine Learning pour la prédiction du risque cardiaque.

Le projet propose différents objectifs métier, notamment la classification des patients à risque, la prédiction d’un score de risque, la classification multi-classe et le clustering de patients similaires.

## Utilisation

### Étape 1 : Entraîner les modèles

Avant de lancer l’application, entraînez et sauvegardez les modèles :

```bash
python train_models.py
```

Cette étape va générer les fichiers suivants :

* `model_BO1.pkl` : modèle de classification binaire
* `model_BO2.pkl` : modèle de régression
* `model_BO3.pkl` : modèle de classification multi-classe
* `model_BO4.pkl` : modèle de clustering
* `scaler_BO1.pkl`, `scaler_BO2.pkl`, `scaler_BO3.pkl`, `scaler_BO4.pkl` : fichiers de normalisation des données
* `features_BO1.pkl`, `features_BO2.pkl`, `features_BO3.pkl`, `features_BO4.pkl` : variables utilisées par les modèles
* `top_5_features_BO3.pkl` : les 5 variables les plus importantes pour BO3
* `cluster_info_BO4.csv` : informations sur les clusters

---

### Étape 2 : Lancer l’application

```bash
streamlit run app.py
```

L’application sera accessible à l’adresse suivante :

```bash
http://localhost:8501
```

## Structure de l’Application

L’application est composée de plusieurs pages :

| Page           | Description                                                              |
| -------------- | ------------------------------------------------------------------------ |
| Page d’accueil | Présentation du projet, des objectifs métier et du pipeline global       |
| BO1            | Classification binaire pour identifier les patients à haut risque        |
| BO2            | Prédiction d’un score continu de risque entre 0 et 100                   |
| BO3            | Classification multi-classe selon trois niveaux : faible, moyen et élevé |
| BO4            | Clustering pour regrouper les patients selon des profils similaires      |

---

## Objectifs Métier

### BO1 : Classification binaire

Identifier précocement les patients asymptomatiques présentant un risque cardiaque élevé.

### BO2 : Régression

Évaluer le risque comportemental à l’aide d’un score continu compris entre 0 et 100.

### BO3 : Classification multi-classe

Classer les patients selon trois niveaux de risque :

* Risque faible
* Risque moyen
* Risque élevé

### BO4 : Clustering

Identifier des groupes de patients similaires afin de mieux comprendre les profils de risque.

# Eco Ressource Backend

Eco Ressource Backend est l'API Spring Boot de la plateforme Eco Ressource, une marketplace B2B orientee economie circulaire. Le projet centralise la gestion des utilisateurs, entreprises, annonces, stocks, reservations, livraisons, dons, evenements solidaires et modules financiers.

L'objectif de la plateforme est de faciliter la reutilisation, l'echange, la donation et la valorisation de ressources entre acteurs professionnels et associatifs, tout en proposant des services complementaires comme la moderation, l'analyse assistee par IA, le suivi logistique, les paiements et la gestion de documents.

## Fonctionnalites principales

- Authentification et securisation des endpoints avec Spring Security et JWT.
- Gestion des utilisateurs, roles, entreprises, transporteurs et associations solidaires.
- Publication, recherche, moderation et gestion des annonces et ressources.
- Gestion des produits, stocks, mouvements d'inventaire et fichiers associes.
- Reservations, commandes de livraison, expeditions et suivi logistique.
- Dons, participations aux evenements et documents d'evenement.
- Modules finance : factures, transactions, financement, escrow, analytics et integrations de paiement.
- Services IA pour la generation de descriptions, l'analyse de defauts et l'aide a la recherche de produits.
- Notifications et integrations externes : email, Discord, Stripe, Konnect, services de geocodage et APIs IA.

## Technologies utilisees

- Java 17
- Spring Boot 3.2.5
- Spring Web, Spring Security, Spring Data JPA, Validation et WebSocket
- MySQL
- Maven
- JWT
- Springdoc OpenAPI / Swagger
- Stripe SDK
- Thymeleaf pour les templates email

# Eco Ressource B2B

Eco Ressource B2B est une application web Angular dediee a la gestion et a la valorisation des ressources entre entreprises. La plateforme centralise les annonces, les stocks, les reservations, les livraisons, les transactions et les evenements afin de faciliter les echanges B2B dans une logique d'economie circulaire.

Le projet propose des espaces separes selon le profil utilisateur : entreprise, transporteur et administrateur. Chaque role dispose d'un tableau de bord et de fonctionnalites adaptees a ses besoins operationnels.

## Objectifs du projet

- Mettre en relation les entreprises autour de ressources, produits ou stocks disponibles.
- Simplifier la publication, la recherche et la gestion des annonces.
- Suivre les stocks, les inventaires, les mouvements et les produits.
- Organiser les reservations, les commandes et les livraisons.
- Donner aux transporteurs un espace pour suivre les trajets, expeditions, bons de livraison et revenus.
- Offrir a l'administrateur une vision globale sur les utilisateurs, annonces, stocks, livraisons, reservations, finances, evenements et actions de solidarite.

- Generation de rapports et consultation des indicateurs financiers.
- Modules complementaires : recherche de produits, chatbot de marche, factures, tresorerie, evenements et solidarite.

### Espace transporteur

- Tableau de bord transporteur.
- Gestion des trajets et expeditions.
- Consultation des bons de livraison.
- Suivi des revenus.

### Espace administrateur

- Gestion des utilisateurs.
- Supervision des annonces, stocks, produits et mouvements de stock.
- Suivi des livraisons, expeditions et reservations.

# TraceDuTest

**ASP.NET Core 8** web API for centralized management of **test traces**, **serial numbers**, **operations**, **log files**, and **users**, in an industrial manufacturing context (quality control and traceability of tests on serialized equipment).

This web portal is dedicated to viewing and administering test data, with **role-based access control** (see the use case diagram below).

## Features (business view)

| Actor | Main use cases |
|-------|----------------|
| **User** | Register, authenticate, view profile, lock the application, browse test traces and statistics |
| **Test manager** | Full *User* scope + **log file management** |
| **Administrator** | Full *User* scope + **serial number management**, **operation management**, **notifications**, **user management** |

## Global use case diagram

![Use case diagram](docs/use-case-global.jpg)

## Data model (conceptual view)

Main entities and relationships are summarized in the following class diagram (logical names: `NumSérie`, `TraceDuTest`, `Opération`, `FichierLog`, `Utilisateur`, `Notification`).

![Class diagram](docs/diagramme-classes-global.jpg)

In the .NET implementation, tables are exposed through **Entity Framework Core** (`TraceDuTestContext`): `User`, `Trace`, `NumeroSeries`, `Operation`, `Fichier`, `Notification`.

## Tech stack

- **.NET 8** — REST API (controllers under `Controllers/TraceController/`)
- **Entity Framework Core 8** + **SQL Server**
- **JWT authentication** (`Microsoft.AspNetCore.Authentication.JwtBearer`)
- **E-mail** (Sendinblue / Brevo via `Sendinblue` package)
- **Spreadsheets / export**: **EPPlus** (`OfficeOpenXml`)
- **Swagger** in the Development environment

## Prerequisites

- [.NET 8 SDK](https://dotnet.microsoft.com/download)
- SQL Server (instance reachable with the connection string set in `appsettings.json`)

## Configuration

1. **Connection string** — In `appsettings.json`, set `ConnectionStrings:DefaultConnection` to your server and `TraceDuTest` database.
2. **JWT** — `JWT:Secret` must **not** be empty: locally, use [user secrets](https://learn.microsoft.com/en-us/aspnet/core/security/app-secrets) (`dotnet user-secrets set "JWT:Secret" "<key at least 64 characters>"`) or the `JWT__Secret` environment variable. Otherwise the application will not start.
3. **E-mail** — Set `Smtp:User`, `Smtp:Password`, `Smtp:From` (and optionally `Smtp:Host` / `Smtp:Port`) in user secrets or a local, non-committed file. If these are empty, outgoing mail is skipped without error. `Sendinblue:ApiKey` remains available for a future API integration if needed.

## Run the API

```bash
dotnet restore
dotnet run --project TraceDuTest.csproj
```

In Development, Swagger UI is available at the URL printed in the console (often `https://localhost:<port>/swagger`).

- `DTO/` — Data transfer objects
